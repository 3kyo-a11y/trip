# 旅行攻略网站 - 需求文档

## 项目概述

开发一个面向中国热门景点的旅行攻略展示网站，为用户提供详细的景点信息、游玩建议、交通指南等内容。

---

## 一、核心功能

### 1. 首页
- 热门景点推荐轮播/卡片展示
- 按地区分类浏览（华东、华北、华南、西南、西北、东北、华中）
- 搜索栏：支持景点名称、城市关键词搜索
- 快速筛选：按季节（春/夏/秋/冬）、预算（经济/中等/高端）、出行方式（亲子/情侣/独行/家庭）

### 2. 景点详情页
每个景点包含以下模块：
- **基本信息**：名称、所在城市、最佳旅游季节、建议游玩时长、门票价格
- **景点介绍**：历史文化背景、特色亮点、评分
- **游玩攻略**：
  - 推荐路线（一日游/两日游/多日游）
  - 必打卡景点列表
  - 拍照机位推荐
- **交通指南**：
  - 如何到达（高铁/飞机/自驾）
  - 当地交通（公交/地铁/打车/租车）
- **住宿推荐**：周边酒店/民宿分级推荐
- **美食攻略**：当地特色美食及推荐餐厅
- **费用预算**：人均消费参考（门票+交通+住宿+餐饮）
- **实用贴士**：注意事项、避坑指南、必备物品清单
- **用户评论**：评分、文字评价、游记分享

### 3. 攻略列表页
- 分页展示各景点攻略卡片
- 排序方式：热度、最新、评分
- 标签筛选：自然风光/人文历史/主题乐园/古镇古村/海滩沙漠

### 4. 收藏与分享
- 用户可收藏感兴趣的攻略
- 支持分享到微信、微博、QQ等社交平台

## 三、技术选型

### 前端
- **框架**：React 18 + TypeScript
- **UI 组件库**：Ant Design 或 shadcn/ui
- **样式方案**：Tailwind CSS
- **路由**：React Router v6
- **状态管理**：Zustand（轻量级）
- **构建工具**：Vite

### 后端（可选，首期可先用静态数据）
- **方案 A**：Node.js + Express + SQLite（轻量部署）
- **方案 B**：纯静态站点（GitHub Pages / Vercel / Netlify）

### 数据
- 初期使用 JSON 文件存储攻略数据
- 后续可扩展为数据库（SQLite / PostgreSQL）

## 五、设计风格

- **整体风格**：清新自然，以绿色/蓝色为主色调，搭配白色背景
- **字体**：中文优先使用系统默认字体栈
- **图标**：使用 Lucide React 图标库
- **响应式**：适配桌面端和移动端
- **图片**：每个景点配 1-3 张高质量封面图（初期可使用占位图）

---

## 六、非功能需求

- 首屏加载时间 < 2 秒
- SEO 友好（SSR 或预渲染，meta 标签完善）
- 无障碍基础支持（aria-label、语义化标签）
- 代码规范：ESLint + Prettier
